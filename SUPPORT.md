# Support

Use GitHub Issues for installation problems, configuration questions, and
behavior that is unclear. Add a `question` label if it is available. If GitHub
Discussions are enabled for the repository, usage questions can go there
instead.

For reproducible bugs, open a bug report and include your operating system,
Python version, TTS backend, whether LLM cleaning was enabled, and clear steps to
reproduce the problem. For feature ideas, open a feature request and describe the
use case rather than only the proposed implementation.

ReadMyPaper is a volunteer-maintained beta project, so support is best-effort.

Do not attach private PDFs, medical documents, PHI, generated audio, cleaned
text, or screenshots with sensitive content to public issues. Use synthetic
examples or describe the relevant behavior instead.
